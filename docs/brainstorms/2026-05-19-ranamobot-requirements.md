# RanamoBot — AI Chat Widget

## Problem Frame

The "AI Counselor" button in the site header currently opens a maintenance modal with
WhatsApp and Book Consultation fallback CTAs. This creates a dead end for visitors who
want instant answers about study abroad — they either have to message a human or leave.

A real AI chatbot (RanamoBot) will give visitors immediate, helpful responses about
Ranamo's services and general study-abroad questions, reducing friction before they
decide to book a consultation. The standalone WhatsApp floating bubble is also replaced —
both human escalation paths are consolidated inside the widget for a cleaner layout.

## Requirements

**Widget & Trigger**
- R1. A single floating chat bubble sits fixed at the bottom-right of every page, always visible above other content (z-index 60 or above, ensuring it appears above the main page layout and any modals).
- R2. The existing standalone `WhatsAppButton.astro` floating bubble is removed. WhatsApp access moves inside the chat widget.
- R3. Clicking the floating bubble opens the chat panel (slides/fades upward).
- R4. The "AI Counselor" button in the desktop header and the mobile nav item both trigger the same widget to open — replacing their current behavior of opening the maintenance dialog.
- R5. The existing `#ai-counselor-dialog` maintenance modal in `src/components/Header.astro` is removed along with its script and style blocks.

**Chat Panel UI**
- R6. The panel sits above the floating bubble, anchored to the bottom-right corner. Max width ~380px on desktop; full-width on mobile (with margin).
- R7. Panel header shows: a sparkle/bot avatar, the name "RanamoBot", a green "online" dot, and a close button (✕).
- R8. A persistent footer inside the panel shows two compact buttons: "Chat on WhatsApp" (opens WhatsApp link) and "Book Consultation" (links to `/contact`). These are always visible regardless of scroll position in the message area.
- R9. Messages scroll within the panel. User messages appear right-aligned; bot messages appear left-aligned with a small bot avatar.
- R10. A typing indicator (three animated dots) is shown while awaiting a bot response.
- R11. The send button and pressing Enter both submit a message. The send button is disabled (and Enter submission suppressed) when the trimmed input value is empty.

**Opening Experience**
- R12. On first open, RanamoBot immediately sends a welcome message: *"Hi! I'm RanamoBot 👋 Ask me anything about studying abroad or how Ranamo Edu can help you."*
- R13. Below the welcome message, four quick-reply chips are displayed:
  - "What countries do you cover?"
  - "How does Ranamo's process work?"
  - "How much does studying abroad cost?"
  - "Scholarship & visa help"
- R14. Clicking a chip submits it as a user message and triggers a bot response. Chips disappear once any message is sent.

**AI Behaviour**
- R15. Bot responses are generated by Google Gemini 1.5 Flash via a Netlify serverless function (`netlify/functions/chat.js`). The Gemini API key is stored as a Netlify environment variable — never in client-side code.
- R16. The system prompt defines RanamoBot as Ranamo Edu's assistant with knowledge of:
  - 7 destination countries: UK, USA, Canada, Australia, Germany, New Zealand, Ireland — including tuition ranges, living costs, entry requirements, intake months, and top universities (sourced from `src/data/destinations.ts`)
  - 6 Ranamo services: University Selection, Application Support, Visa Assistance, Scholarship Guidance, Test Preparation, Pre-Departure Support (sourced from `src/data/services.ts`)
  - General study-abroad topics: IELTS/TOEFL/PTE/GRE/GMAT, SOP writing, LOR, financial documentation, PR pathways
- R17. When a query requires personal profile analysis or a decision specific to the user's situation, the bot encourages booking a free consultation rather than speculating.
- R18. The bot does not fabricate admission statistics, visa approval rates, or fees beyond the ranges defined in the data files.

**Session Behaviour**
- R19. Chat history persists across Astro view transitions (soft navigations) within the same browser session. History resets only on full page refresh or tab close — no localStorage required. The chat panel component must use `transition:persist` to survive Astro view transitions.
- R20. The conversation history (user + assistant turns) is sent with each API request so the bot maintains context within a session.

## Success Criteria
- A visitor can open the widget, ask a general study-abroad question, and receive a relevant answer within 3 seconds on a normal connection.
- The WhatsApp and Book Consultation CTAs are reachable within the chat panel without closing it.
- The header "AI Counselor" button and the floating bubble both open the same widget correctly on desktop and mobile.
- The Gemini API key is not present in any client-side bundle or network request.

## Scope Boundaries
- No user login or persistent chat history across sessions.
- No multilingual support — English only.
- No file or document upload.
- No CRM/lead-capture integration (the WhatsApp and contact links serve this purpose).
- No streaming token-by-token response — single response delivery is sufficient.
- No Astro SSR mode change — the site remains `output: 'static'`; the Netlify Function handles the server-side API call.

## Key Decisions
- **Google Gemini 1.5 Flash over Groq/OpenRouter**: Gemini's free tier is the most generous (1M tokens/day) and has been stable for over a year. No trial credits — genuinely free.
- **Netlify Function over client-side API call**: Keeps the API key off the client bundle. Netlify's free tier allows 125,000 function invocations/month — more than sufficient.
- **Floating widget over modal chat**: Industry standard pattern (Intercom/Crisp/Drift); always accessible without navigating to the header; avoids two competing floating elements.
- **WhatsApp absorbed into widget**: Removes the competing bottom-right bubble and makes human escalation a natural step within the chat rather than a separate UI element.
- **Chat history in-memory only**: localStorage adds complexity and privacy considerations with no meaningful benefit for a lead-gen chatbot.

## Dependencies / Assumptions
- A Google Gemini API key must be obtained (free at aistudio.google.com) and added as `GEMINI_API_KEY` in Netlify environment variables before deployment.
- The Netlify build/deploy pipeline is already working (`netlify.toml` confirmed). Netlify Functions require no additional plugin for this use case.
- `src/data/destinations.ts` and `src/data/services.ts` are the authoritative source for Ranamo-specific facts embedded in the system prompt.
- The Netlify Function must restrict `Access-Control-Allow-Origin` to the production domain (`ranomoedu.com`) and localhost for development — a wildcard CORS policy would allow any site to proxy calls through it.
